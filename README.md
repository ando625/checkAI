
# CheckPoint AI 〜AIによる業務報告の要約と分析ツール〜

## アプリの概要

このアプリを一言でいうと、**「AIがマネージャーの代わりをしてくれる、お仕事報告管理ツール」**です。
単に業務報告を保存するだけでなく、Python（FastAPI）とGemini AIを連携させ、投稿内容から「仕事の危険度」を自動でスコア化し、要約まで行うことで、チームの状況をひと目で把握できる仕組みを実現しました。

### 開発の背景

「あれ、あの件どうなったっけ？」「さっき言ったよね？」といった、職場での**「言った・言わない」のトラブルをなくしたい**と思ったのが開発のきっかけです。

また、忙しい時に口頭で報告されると、作業を一度止めなければならず、お互いに集中力が切れてしまうことがあります。このアプリなら、**「報告する側は好きな時に送れる」「確認する側は手が空いた時にチェックできる」**という、自分のペースで仕事ができる環境を作れます。

さらに、溜まった報告を一つずつ読むのは大変ですが、**AIがパッと要約してくれたり、仕事の調子をグラフにしてくれる**ことで、「今はどこに力を入れるべきか」がすぐに分かり、チーム全体の仕事がもっとスムーズに進むようになることを目指しました。

これまで学んできたPHP/Laravelの技術に加え、フロントエンドに**Next.js (App Router)**、AI解析に**Python**を導入。「人間が全部読まなくても、AIがリスクをあぶり出してくれる親切な設計」を目指して開発しました。

---

### こだわったポイント


**1. 「言った・言わない」をなくす確実な記録**
*   口頭だと忘れてしまいがちな報告を、文字と時間でしっかり残します。
*   後からいつでも見返せるので、「あの時どう決まったっけ？」という確認作業がなくなります。

**2. 集中力を切らさない「非同期」のコミュニケーション**
*   口頭報告のように相手の手を止める必要がありません。
*   コメント機能を使って、お互い好きなタイミングで内容の確認やアドバイスができるので、自分の作業に集中できる時間を増やせます。

**3. AIによる「仕事の健康診断」**
*   溜まったデータをAIが分析して、「今、仕事が捗っているか」「改善すべき点はあるか」をまとめてくれます。
*   自分で計算しなくても、AIが客観的な数字で教えてくれるので、次の仕事の作戦が立てやすくなります。

**4. 「一瞬で状況がわかる」リスク検知ロジック**
* 投稿された文章をPythonが解析し、「順調」「要確認」「リスクあり」の3段階で自動仕分けします。
* 深刻なトラブルを抱えた報告には赤いアラートを表示し、管理者が**「今、誰を助けるべきか」を1秒で判断できる**ように工夫しました。

**5. 「読む時間を9割削る」AI要約機能**
* 長い報告文も、AIが重要なポイントを3行程度に要約します。
* 忙しい朝の進捗確認でも、要約をパッと眺めるだけでチーム全体の動きを把握できる「タイパ（タイムパフォーマンス）」にこだわりました。

**6. 「直感的に推移を掴む」分析グラフ**
* 毎日の感情・進捗スコアを蓄積し、Recharts を使ってグラフ化しました。
* 「先週から急にスコアが下がっている」といった目に見えないチームの予兆を、客観的な数字でキャッチできるようにしています。

**7.「止まらない」快適な操作感**
* Next.jsの特性を活かし、画面全体を読み込み直さずに投稿やコメントができるスムーズな動きを追求しました。
* AIの解析待ちの間もスケルトンスクリーンを出すなど、ユーザーを待たせないUI設計を心がけています。
* 分析データのAI詳細要約は時間がかかるので下部におき、上部のデータは比較的早く表示されるようにuseEffectを分けて設定しました。
* AI詳細分析は３０分後にリロードした場合再取得設定。




### 苦労した点

* **3つの言語・技術の連携（Next.js × Laravel × Python）**
* Laravelを「司令塔」として、フロントからのリクエストを受け取り、Pythonへ解析を依頼し、結果をDBへ保存するというデータの流れを制御するロジックの構築が最大の挑戦でした。
* **Dockerによる多層環境の構築**
* フロント、バック、DBに加え、AI解析用のPython環境までを一つのDockerネットワークで共存・通信させる設定に苦労しましたが、この経験でインフラへの理解が深まりました。
* **保守性を意識した「型定義（TypeScript）」の徹底**
* これまではコンポーネント内に直接データを書いてしまうこともありましたが、今回は実務を意識し、APIから返ってくるデータ（Report, Comment, User等）に対して厳密な `interface` や `type` を定義しました。
* 最初は型エラーに苦戦しましたが、型をしっかり決めることで「どんなデータが流れてくるか」が明確になり、結果としてバグの少ない開発ができました。
* **ロジックの共通化（カスタムフックの活用）**
* 「データの取得」「投稿処理」「削除処理」などの複雑なロジックをコンポーネントから切り離し、`useReports` や `useAuth` といったカスタムフックにまとめました。
* これにより、コンポーネント（見た目）がスッキリし、どこで何をしているかが一目でわかる「読みやすいコード」を目指しました。

## システム連携図（AI分析の流れ）

### 報告投稿時の「ロジック」

| ステップ | 担当 | 役割 |
| --- | --- | --- |
| **1. 報告する** | Next.js | ユーザーが業務内容を入力 |
| **2. 依頼を投げる** | Laravel | データを保存し、Python(FastAPI)へ解析依頼 |
| **3. AI解析** | Python (Gemini) | 内容をスコア化（0〜100点）し、要約文を作成 |
| **4. 可視化** | Next.js | スコアに基づき、バッジの表示やグラフを即座に更新 |

## 主な機能

1. **AI自動スコアリング＆要約**
投稿した瞬間にAIが内容を分析。ポジティブ・ネガティブ度を数値化し、要約文を生成します。
2. **ダッシュボード（分析表示）**
チーム全体の平均スコアや、リスク報告の件数をカード形式で表示。グラフにより長期的な傾向を分析できます。
3. **ステータスバッジ機能**
AIが判定した状況（順調・要確認・リスクあり）に応じて、一覧画面にカラーバッジを表示。視覚的な優先順位付けを助けます。
4. **コメント＆コミュニケーション**
各報告に対して、その場でフィードバックや質問を送れます。口頭報告とは違い、**「報告者は好きな時に送り、確認者は手が空いた時に返信できる」**ため、お互いの作業の手を止めることなく、集中力を維持したままスムーズに内容確認が行えます。
5. **AI詳細分析（インサイト）**
蓄積されたデータから、AIが「来週の予測」や「現在のトピック（頻出ワード）」を抽出してアドバイスを提示します。

## セットアップ方法

1. **リポジトリをクローン**
```bash
git clone git@github.com:ando625/checkAI.git
cd checkAI
```

2. **Dockerコンテナの起動**
```bash
docker compose up -d --build

```

3. snsappからフロントエンドnextで実行
```
snsapp
├──  laravel(backend)
└──  next(frontend) ← ココ
```

```
npm install
```




4. プロジェクトのルートphp上で実行
```
docker compose exec php bash
```



5. **環境設定ファイルの準備**
```bash
cp .env.example .env

```

※ .env ファイルを開き、データベースの設定を以下のように書き換えてください：

```
DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=laravel_db
DB_USERNAME=laravel_user
DB_PASSWORD=laravel_pass
```


6. **ライブラリのインストール**

```bash
composer install
```




7. **アプリケーションキーの生成とデータベース、シンボリックリンクの準備**
```bash
php artisan key:generate
php artisan migrate:fresh --seed
php artisan storage:link
```
